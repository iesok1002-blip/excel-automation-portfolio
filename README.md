PythonによるExcel業務自動化ポートフォリオ


## 各スクリプトの解説

calculate_tax.py：　Pythonの基本を学習
combine_excel.py：　複数ファイル検索とデータ結合
filter_data.py：　単一条件でのデータ抽出と日付書式設定
filter_complex_data.py：　AND条件での複数条件抽出
filter_or_data.py：　OR条件での複数条件抽出
calculate_groupby.py：　グループ別集計
calculate_pivot.py：　クロス集計表（ピボットテーブル）作成


## 学習プロセスと課題

課題1：開発環境の全体像の理解
　今回の学習で最初につまづいたのが、ターミナル操作、仮想環境、ライブラリ管理という一連の流れでした。なぜGUIではなくターミナルを使うのか、なぜ仮想環境を作る必要があるのか、しかし、これらがIT実務の基本であること、将来ライブラリ同士が競合する可能性があるという重要な必要意義について学びました。

課題2：生成AI（Gemini）利用の弊害と対策
　Geminiとの対話を通じて体系的に学習を進めていますが、Geminiが最適解を提示するため作業がスムーズに進みすぎてしまい、表面上の学習になってしまう可能性を課題と感じました。対策として、作成したコードは数日おいてから自力で書き、知識の定着を図っています。

課題3：課題3：アルゴリズム理解の重要性
　Pythonの文法については学習開始当初より理解が深まりましたが、やはり文法のみ覚えるのでは自力での作成には時間がかかりすぎると感じました。そのため、「何をしたいか」を実現するためのアルゴリズムの理解を最近は進めています。


## 生成AIの活用について

Pythonの学習においては、「Gemini」を活用しました。
単にコードを生成させるだけでなく、概念の根本的な理解や、発生したエラーの解決、コード改善の提案を受けるために使用しました。
AIが提示したコードに対し、その意味を私が再度質問して理解を深める、というペアプログラミングのような形で学習を進めました。
